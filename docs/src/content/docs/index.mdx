Hostwright is a **Mac-native desired-state control plane** for [Apple container](/docs/concepts/apple-container-boundary/) workloads.

You declare local services in a restricted [`hostwright.yaml`](/docs/reference/manifest/) manifest. Hostwright validates it, computes a deterministic non-mutating [plan](/docs/concepts/reconciliation/), observes the runtime through a single typed [adapter boundary](/docs/concepts/runtime-adapter/), persists state to an explicit SQLite path you choose, and mutates the runtime only through narrow, hash-confirmed gates.

It is built for one machine: an Apple silicon Mac running macOS 26 or newer with Apple container installed. It is not a cluster orchestrator, and it does not try to be.

<Aside type="caution" title="Source-only alpha">
Hostwright `v0.1.0-alpha.1` is a source-only alpha. It is not production ready.
There are no binary downloads, installers, Homebrew formulae, signed or notarized
artifacts. You [build it from source](/docs/getting-started/install-from-source/).
Read [Limitations](/docs/reference/limitations/) before relying on any behavior.
</Aside>

## What works today

- `hostwright init`, `validate`, and `plan` — restricted manifest parsing, fail-closed validation, and deterministic non-mutating planning with a plan hash.
- `hostwright status --state-db <path>` — live runtime observation through the RuntimeAdapter with event and snapshot persistence.
- `hostwright apply --state-db <path> --confirm-plan <hash>` — exactly **one** confirmed action per invocation: create a missing service, or a restart-policy-allowed managed start or managed restart.
- `hostwright logs`, `events`, `recovery`, `diagnostics` — bounded, redacted, local-only observability from the explicit state database.
- `hostwright cleanup` — dry-run classification first, then token-confirmed deletion of exact Hostwright-owned stopped/created/exited containers only.
- `hostwrightd --foreground` — a foreground reconciliation loop that observes, plans, runs loopback health probes, and records events. It never mutates the runtime.
- `keychain://` [secret references](/docs/concepts/secrets-and-keychain/) in `secretEnv`, backed by an opt-in noninteractive read-only macOS Keychain reader.
- `hostwright doctor` — safe local checks and resource intelligence, with unmeasured dimensions reported as unmeasured.

Everything else — general lifecycle management, multi-action apply, unattended daemon mutation, DNS, tunnels, Kubernetes or Docker API compatibility, Compose parity — is a non-goal or not implemented. The [Limitations](/docs/reference/limitations/) page is the authoritative list.
